# 📖 Глоссарий предметной области

> Версия: 1.0 | Дата: 2025-01 | Автор: Марина

| Термин | Определение | Пример |
|--------|------------|--------|
| **Ковенанта (Covenant)** | Обязательство заемщика, закреплённое в кредитном договоре, поддерживать определённые финансовые показатели или выполнять действия | «Debt/EBITDA ≤ 3.0 на конец каждого квартала» |
| **Финансовая ковенанта** | Ковенанта, основанная на расчётных финансовых показателях | EBITDA, Current Ratio, Debt/Equity |
| **Нефинансовая ковенанта** | Ковенанта, основанная на событиях или юридических фактах | Отсутствие судебных исков > ₽50 млн, сохранение залога |
| **Cross-default** | Перекрёстный дефолт: нарушение ковенанты по одному договору автоматически считается нарушением по всем договорам группы | Компания А нарушила ковенанту по кредиту в банке X → дефолт по кредиту в банке Y |
| **EBITDA** | Прибыль до вычета процентов, налогов и амортизации. Ключевой показатель для ковенант | Выручка ₽1 млрд − Опер. расходы ₽700 млн = EBITDA ₽300 млн |
| **Транш** | Часть кредитной линии, выдаваемая заёмщику в определённый момент | Кредитная линия ₽5 млрд, выдано траншами: ₽2 млрд в Q1, ₽1 млрд в Q2 |
| **Лимит** | Максимальная сумма задолженности, доступная заёмщику | Общий лимит на группу компаний: ₽10 млрд |
| **Алерт (Alert)** | Системное уведомление о потенциальном или фактическом нарушении ковенанты | «Ковенанта Debt/EBITDA нарушена: факт 3.5 > порог 3.0» |
| **EWS (Early Warning Signal)** | Сигнал раннего предупреждения — индикатор ухудшения кредитного качества до формального нарушения | Падение выручки на 30% за квартал |
| **False Positive** | Ложное срабатывание алерта — ковенанта рассчитана неверно или данные неактуальны | ФНС прислала старые данные, фактически налог уплачен |
| **Maker-Checker** | Принцип разделения полномочий: один сотрудник создаёт/изменяет, другой подтверждает | Риск-аналитик создаёт алерт → Риск-менеджер подтверждает |
| **Acceleration** | Право банка потребовать досрочного погашения всего долга при существенном нарушении ковенант | При Debt/EBITDA > 5.0 банк имеет право потребовать возврат ₽3 млрд |
| **Grace Period** | Льготный период, в течение которого заёмщик может устранить нарушение без штрафных санкций | 30 дней на восстановление Current Ratio |
